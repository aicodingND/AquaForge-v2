Data Flow
=========

1. User Interaction
   └─> Component (UI)
       └─> State Event Handler
           └─> Service Layer
               ├─> Cache Check
               ├─> Core Logic
               └─> Return Results
                   └─> Update State
                       └─> Re-render UI

2. File Upload Flow
   └─> upload.py (Component)
       └─> State.handle_upload()
           ├─> Validation (file type, size)
           ├─> Sanitization
           ├─> File Save
           └─> PDF Parsing
               └─> Update RosterState

3. Optimization Flow
   └─> optimize.py (Component)
       └─> State.run_optimization()
           └─> OptimizationService.predict_best_lineups()
               ├─> Cache Check (90% hit rate)
               ├─> Validation
               ├─> Core Optimizer
               │   ├─> Heuristic (Simulated Annealing)
               │   └─> Gurobi (Exact Solution)
               └─> Cache Store
                   └─> Update OptimizationState

Key Benefits
============

✓ Separation of Concerns: UI, State, Services, Core Logic
✓ Reusability: Shared components and utilities
✓ Testability: Modular design, easy mocking
✓ Performance: Caching, async processing
✓ Maintainability: Clear structure, type hints
✓ Scalability: Easy to add new features
✓ Developer Experience: Better IDE support, documentation
