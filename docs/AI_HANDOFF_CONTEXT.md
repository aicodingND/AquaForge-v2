## 1. System Architecture (Railway Deployment)

- **Platform:** Railway (Docker-based)
- **Web Server:** Caddy (v2.8+) acts as Reverse Proxy & Static File Server.
  - **Config:** `/app/Caddyfile`
  - **Frontend:** Served from `/app/.web/build/client` (Static Export)
  - **Backend:** Proxied to `localhost:8000` (`reflex run --backend-only`)
  - **Health Check:** `/health` endpoint (handled by Caddy directly)
- **Frontend Construction:**
  - **Build Time:** `reflex export --frontend-only` runs in Dockerfile `builder` stage.
  - **Runtime Patching:** `start.sh` uses `sed` to replace `RAILWAY_DOMAIN_PLACEHOLDER` with `RAILWAY_PUBLIC_DOMAIN`.
  - **Reasoning:** Prevents "mangled UI" issues caused by hardcoded `localhost` or stale cached URLs.

## 2. Critical Logic Flows

### A. Meet Alignment (Score Inflation Fix)

- **Problem:** "158-294" score inflation caused by PDFs containing multiple meets (e.g., season history).
- **Solution:** `meet_alignment_service.align_meet_data`
- **Location:** `swim_ai_reflex/backend/services/meet_alignment_service.py`
- **Integration:** Called in `OptimizationState.run_optimization` _before_ prediction.
- **Mechanism:**
  1. **Strict:** Checks `opponent` column for name match (Deterministic).
  2. **Fuzzy:** Checks event intersection if opponent column missing (Adaptive).
- **Logging:** Explicitly logs "Exact Meet Match" vs "Fuzzy Match Used".

### B. Score Validation

- **Location:** `swim_ai_reflex/backend/services/score_validation_service.py`
- **Trigger:** runs _after_ optimization in `OptimizationState`.
- **Rules:** Checks for duplicate swimmers, excessive entries, and impossible scores (>160pts/team).

## 4. Known Constraints & "Gotchas"

1. **Frontend/Backend Port:** Caddy listens on `$PORT` (Railways default), proxies internal `:8000`. Frontend is NOT on `:3000` in prod.
2. **Assets Path:** Do NOT add a generic `handle /assets/*` to Caddyfile. It masks the real `_static/assets` generated by Reflex.
3. **PDF Parsing:** The caching mechanism relies on MD5 hashing of the file. If verifying "new" logic, ensure the PDF is actually re-processed or cache is cleared.
4. **Scraper Dependencies:** `web_scraper_service.py` has lazy imports. It allows `playwright` code to exist without crashing the app if the package isn't installed.
